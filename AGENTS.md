# SkinCoach — инструкции для агента

Проект: `C:\Users\Design\Desktop\skincoach`

## Анализ недавних провалов

Ниже — почему предыдущие попытки починить/добавить веб часто не работали у пользователя, хотя у агента в тестах всё было ОК.

### 1. Рассинхрон версий и кеш
- Проблема: HTML загружался из кеша браузера, а JS — новый, или наоборот. FastAPI StaticFiles не сбрасывает клиентский кеш автоматически.
- Следствие: у пользователя были видны старые кнопки с новой логикой, обратная ситуация, или вовсе неработающие обработчики.
- Решение: держать фронт в одном файле, добавлять `Cache-Control: no-cache`, использовать версионирование ресурсов (`?v=3`), после изменений перезапускать сервер и просить пользователя `Ctrl+F5`.

### 2. Внешние CDN-зависимости
- Проблема: Tailwind CSS подгружался с `cdn.tailwindcss.com`. У агента в тестовом Chromium CDN грузился; у пользователя мог быть заблокирован, медленный или кешированный старый вариант.
- Следствие: стили ломались, кнопки не кликались, текст мог перекрываться.
- Решение: не использовать внешние CDN для MVP. Всё critical CSS — inline.

### 3. Разделение HTML и JS
- Проблема: `index.html` + `app.js` — два файла. Если один из них сохранился в неправильной кодировке (UTF-16LE вместо UTF-8) или закешировался, сайт ломается.
- Следствие: кракозябры вместо кириллицы, не находятся обработчики, ошибки 404.
- Решение: для простых SPA держать HTML, CSS и JS в одном `index.html`. Это устраняет проблемы рассинхрона и кодировки между файлами.

### 4. Кодировка файлов
- Проблема: при восстановлении через `git show` файл сохранялся в UTF-16LE. FastAPI отдавал его как UTF-8, и браузер показывал мусор.
- Следствие: весь кириллический текст превращался в квадраты.
- Решение: всегда сохранять `.html`, `.js`, `.css` в UTF-8 без BOM. Перед коммитом проверять первые байты (`b'\xef\xbb\xbf'` — BOM, нежелателен).

### 5. Запуск uvicorn из неправильной директории
- Проблема: `main.py` импортирует `config` и `database` относительно `web/backend/`. Если запускать `uvicorn web.backend.main:app` из корня проекта, Python не находит `config.py`.
- Следствие: сервер не стартует.
- Решение: запускать из `web/backend/` командой `uvicorn main:app --host 127.0.0.1 --port 8000`.

### 6. Недостаточное тестирование "как пользователь"
- Проблема: агент проверял `curl` или запускал headless-браузер, но не воспроизводил реальные клики, вкладки и перезагрузки.
- Следствие: баги интерфейса обнаруживал только пользователь.
- Решение: обязательно тестировать полный сценарий в реальном браузере (upload → analyze → result → program tab → next day → new photo) и смотреть скриншоты.

### 7. Неправильный путь к `.env`
- Проблема: `.env` находится в корне `C:\Users\Design\Desktop\skincoach`, а команда запуска сервера выполнялась из `web/backend/`. Попытки читать `..\.env` работают только если cwd правильный.
- Решение: указывать абсолютный путь к `.env` или запускать сервер через скрипт, который сам поднимается из корня проекта.

### 8. Несколько Python-окружений
- Проблема: torch/torchvision есть только в venv Hermes, а в `web/backend/.venv` их нет. ML-сервис нужно запускать на том интерпретаторе, где есть `torch`.
- Следствие: ML-сервис падает с `ModuleNotFoundError`.
- Решение: веб-бэкенд — `web/backend/.venv`, ML-сервис — Hermes venv (`C:\Users\Design\AppData\Local\hermes\hermes-agent\venv`).

## Обязательные шаги при изменении веб-части

1. Проверить, что файл сохранён в UTF-8:
   ```powershell
   [System.Text.Encoding]::UTF8.GetString([System.IO.File]::ReadAllBytes("web/backend/static/index.html")) | Out-Null
   ```
2. Перезапустить веб-сервер (`uvicorn main:app` из `web/backend/`).
3. Проверить статус ML-сервиса на `http://localhost:8001/`.
4. Открыть `http://localhost:8000/static/index.html` в реальном браузере.
5. Пройти полный сценарий:
   - клик по зоне загрузки,
   - выбор файла,
   - кнопка «Начать анализ»,
   - появление результата,
   - вкладка «Программа»,
   - кнопка «Следующий день»,
   - кнопка «Новое фото».
6. Сделать скриншот результата и убедиться, что текст на русском читаемый.
7. Сказать пользователю нажать `Ctrl+F5`, если он видит старую версию.

```
Terminal 2 (Hermes venv, из web/ml_service/):
  C:\Users\Design\AppData\Local\hermes\hermes-agent\venv\Scripts\python.exe -m uvicorn main:app --host 127.0.0.1 --port 8001
```

## Главное правило

Не доверяй только `curl`/консоли. Если пользователь говорит, что "не работает", открывай браузер, кликай по настоящим элементам и смотри скриншоты. То, что работает у агента в тестовом окружении, ещё не значит, что работает у пользователя.

## Что НЕ трогать без явного запроса

- `bot.py`
- `inference.py`
- `.env`
- `config.yaml` (если появится)
- `class_map.json` — менять только если точно знаешь, что делает (3 класса: melanoma, nevus, other).
