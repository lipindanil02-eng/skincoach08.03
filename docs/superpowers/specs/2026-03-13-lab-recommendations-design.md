## Цель

После фото-диагностики бот рекомендует персонализированный список анализов исходя из диагноза. Пользователь может ввести результаты (текстом или фото бланка), и бот интерпретирует их, давая более точные рекомендации. В 4-ю неделю программы бот напоминает о сдаче/обновлении анализов.

---

## Выбранный подход: Гибрид

- **Список анализов:** хардкод (базовый для всех + персонализированный по диагнозу)
- **Интерпретация результатов:** LLM через `prompts/4b_labs.txt`
- **Ввод:** фото бланка (OCR через vision модель) или текстом вручную

## Флоу

```
ФОТО ПОЛУЧЕНО
    ↓
8-слойный анализ (существующий)
    ↓
[S_QUESTIONS] — уточняющие вопросы (существующий)
    ↓
Диагноз сохранён → отправить сообщение с анализами:
  "🔬 По диагнозу [X] рекомендую сдать:
   Базовые: ОАК, СОЭ, CRP, D, цинк, IgE, гистамин, ...
   Дополнительно для [диагноз]: ..."
    ↓
"Есть уже готовые анализы? Пришли фото бланка или напиши значения (или /skip)"
    ↓
[S_LABS] — новое состояние
  ├─ Фото → vision модель OCR → извлекаем значения → сохранить labs_raw
  ├─ Текст → сохранить labs_raw
  └─ /skip или "пропустить" → выйти из S_LABS, продолжить программу
    ↓
4b_labs.txt → LLM интерпретирует значения в контексте диагноза
→ Уточнённые рекомендации отправляются пользователю
→ Переход в [S_ACTIVE]

НЕДЕЛЯ 4 (день 22, начало):
  ├─ labs уже есть → "Прошло 3 недели, давай обновим анализы? Пришли новые результаты"
  └─ labs нет → "Настало время сдать анализы! Вот список для твоего диагноза: ..."
               → войти в S_LABS

КОМАНДА /labs (в любое время):
  → Войти в S_LABS для ввода/обновления анализов
```

## Изменения в коде

### `bot.py`

1. **Константы:** добавить `S_LABS = "labs"`, `LABS_BASE`, `LABS_BY_DIAGNOSIS`
2. **`gu()` (get_user):** добавить поля `labs`, `labs_raw`, `labs_submitted_at` в дефолт
3. **После `S_QUESTIONS`:** после сохранения диагноза — отправить список анализов + перевести в `S_LABS`
4. **Обработчик фото в `S_LABS`:** вызов vision модели для OCR, сохранить в `labs_raw`, вызвать интерпретацию
5. **Обработчик текста в `S_LABS`:** сохранить как `labs_raw`, вызвать интерпретацию
6. **`/skip` в `S_LABS`:** выход без ввода
7. **Неделя 4 (в `handle_nextday`):** проверить день и наличие `labs`, отправить напоминание/запрос
8. **Команда `/labs`:** вход в `S_LABS` в любое время

### `prompts/4b_labs.txt` (новый файл)

Промпт для LLM интерпретации анализов:
- Входные данные: диагноз, labs_raw, имя пользователя, длительность симптомов
- Задача: интерпретировать каждый показатель, выделить отклонения от нормы, скорректировать рекомендации
- Тон: понятный, без медицинского жаргона, с дисклеймером "не замена врачу"

## Примеры сообщений бота

### Рекомендация сдать анализы (после фотоанализа)
```
🔬 По твоему диагнозу рекомендую сдать:

📋 Базовые (важны для всех):
• ОАК (общий анализ крови)
• СОЭ и CRP — маркеры воспаления
• Витамин D
• Цинк
• IgE общий
• Гистамин
• Копрограмма
• Анализ на паразитов (лучше 3 раза)
• Дисбактериоз
• Ревматоидный фактор (RF)

⚠️ Дополнительно для акне:
• ДГЭАС, тестостерон, ЛГ/ФСГ (гормоны)
• Глюкоза, инсулин

Уже есть готовые анализы? Пришли фото бланка или напиши значения — дам более точные рекомендации.
Нет анализов пока — напиши /skip
```

### Напоминание в 4-ю неделю
```
🔬 Неделя 4 — время анализов!

Ты прошёл 3 недели программы. Теперь важно сдать анализы чтобы оценить прогресс и скорректировать план.

Вот список для твоего диагноза: [список]

Пришли результаты — интерпретирую и обновлю рекомендации.
```

## Ограничения и дисклеймеры

- Рекомендации по анализам носят информационный характер, не заменяют консультацию врача
- OCR фото бланка может ошибаться — пользователь может скорректировать текстом
- При диагнозе melanoma — явный призыв обратиться к дерматологу немедленно
