## Цель

После фото-диагностики бот рекомендует персонализированный список анализов исходя из диагноза. Пользователь может ввести результаты (текстом или фото бланка), и бот интерпретирует их, давая более точные рекомендации. В 4-ю неделю программы бот напоминает о сдаче/обновлении анализов.

---

## Выбранный подход: Гибрид

- **Список анализов:** хардкод (базовый для всех + персонализированный по диагнозу)
- **Интерпретация результатов:** LLM через `prompts/4b_labs.txt`
- **Ввод:** фото бланка (OCR через vision модель) или текстом вручную

## Данные

### Базовый список (все пользователи)

```python
LABS_BASE = [
    "ОАК (общий анализ крови)",
    "СОЭ и CRP (воспаление)",
    "Витамин D",
    "Цинк",
    "IgE общий",
    "Гистамин",
    "Копрограмма",
    "Анализ на паразитов (3-кратный)",
    "Дисбактериоз",
    "Ревматоидный фактор (RF)",
]
```

### Нормализация диагноза

Поле `u["diagnosis"]` содержит свободный текст от LLM (напр. `"акне"`, `"меланома"`).
Для lookup в `LABS_BY_DIAGNOSIS` используется вспомогательная функция `normalize_diagnosis(text)`,
которая приводит к ключу через простой маппинг ключевых слов:

```python
DIAGNOSIS_NORM = {
    "меланом": "melanoma",
    "melanoma": "melanoma",
    "невус": "nevus",
    "родинк": "nevus",
    "nevus": "nevus",
    "акне": "acne",
    "прыщ": "acne",
    "угр": "acne",
    "acne": "acne",
    "атоп": "atopy",
    "экзем": "atopy",
    "atopy": "atopy",
    "себоре": "seborrhea",
    "seborrhea": "seborrhea",
}

def normalize_diagnosis(text: str) -> str:
    t = text.lower()
    for kw, key in DIAGNOSIS_NORM.items():
        if kw in t:
            return key
    return "other"
```

### Персонализированные добавки по диагнозу

```python
LABS_BY_DIAGNOSIS = {
    "melanoma":  ["Онкомаркеры (LDH, S100B)", "Биопсия (консультация дерматолога)"],
    "nevus":     ["Дерматоскопия (консультация)"],
    "acne":      ["ДГЭАС, тестостерон, ЛГ/ФСГ (гормоны)", "Глюкоза, инсулин (сахар)"],
    "atopy":     ["Специфические IgE (аллергопанель)", "Панель пищевой непереносимости"],
    "seborrhea": ["ТТГ, Т3, Т4 (щитовидная)", "Ферритин"],
    "other":     ["Ферритин", "ТТГ"],
}
```

### Хранение в history.json

```json
{
  "labs_raw": "Витамин D = 18, цинк = 7.2",
  "labs_submitted_at": "2026-03-13T10:00:00"
}
```

Только `labs_raw` (строка) сохраняется. Структурированный парсинг в `labs{}` — не делается,
`labs_raw` напрямую передаётся в промпт `4b_labs.txt`.
`labs_submitted_at` используется в неделю 4 для определения: "анализы уже есть или нет".

## Флоу

```
ФОТО ПОЛУЧЕНО
    ↓
8-слойный анализ (существующий)
    ↓
[S_QUESTIONS] — уточняющие вопросы (существующий)
    ↓
Диагноз сохранён (u["diagnosis"] = free-text от LLM)
normalize_diagnosis(u["diagnosis"]) → diagnosis_key
→ отправить сообщение со списком анализов:
  "🔬 По диагнозу [X] рекомендую сдать:
   Базовые: ОАК, СОЭ, CRP, D, цинк, IgE, гистамин, ...
   Дополнительно: [LABS_BY_DIAGNOSIS[diagnosis_key]]"
u["state"] = S_LABS
    ↓
"Есть уже готовые анализы? Пришли фото бланка или напиши значения
(например: D=18, цинк=7.2). Нет — просто напиши 'пропустить'"
    ↓
[S_LABS] — обработка входящего сообщения:

  ФОТО:
    ├─ handle_photo видит state == S_LABS → ветка OCR (НЕ диагностика кожи)
    ├─ vision модель: "Извлеки все числовые показатели из этого бланка анализов"
    ├─ результат → u["labs_raw"], u["labs_submitted_at"]
    └─ вызов интерпретации (4b_labs.txt) → ответ пользователю → S_ACTIVE

  ТЕКСТ (содержит цифры или "=" или ":"):
    ├─ u["labs_raw"] = text, u["labs_submitted_at"] = now
    └─ вызов интерпретации (4b_labs.txt) → ответ пользователю → S_ACTIVE

  ТЕКСТ "пропустить" / "skip" / "нет":
    ├─ без сохранения, без интерпретации
    └─ u["state"] = S_ACTIVE, "Хорошо, продолжай программу!"

  ЛЮБОЙ ДРУГОЙ ТЕКСТ (непонятное сообщение):
    └─ "Пришли фото бланка, напиши значения (D=18, цинк=7) или 'пропустить'"

НЕДЕЛЯ 4 (в cmd_next, когда u["day"] == 22):
  ├─ labs_raw не None → "Прошло 3 недели! Обнови анализы для точных рекомендаций.
  │                      Пришли новые результаты." → u["state"] = S_LABS
  └─ labs_raw is None → "Настало время сдать анализы! Вот список: [список]
                         Пришли результаты — интерпретирую." → u["state"] = S_LABS

КОМАНДА /labs (в любое время):
  ├─ state в (S_NAME, S_DUR, S_TRIED, S_PHOTO, S_QUESTIONS):
  │    → "Сначала заверши первичную диагностику (пришли фото кожи)"
  └─ иначе → отправить список анализов для текущего диагноза
             → u["state"] = S_LABS, "Пришли анализы:"
```

## Изменения в коде

### `bot.py`

1. **Константы:** добавить `S_LABS = "labs"`, `LABS_BASE`, `LABS_BY_DIAGNOSIS`, `DIAGNOSIS_NORM`
2. **Функция `normalize_diagnosis(text)`:** keyword-маппинг → diagnosis_key
3. **`gu()` (get_user):** добавить поля `labs_raw: None`, `labs_submitted_at: None` в дефолт
4. **`handle_photo`:** добавить ветку `if u["state"] == S_LABS` — вызов OCR вместо диагностики кожи
5. **`handle_text`:** добавить ветку `elif u["state"] == S_LABS` — обработка ввода анализов
6. **После `S_QUESTIONS` (в `handle_photo`):** отправить список анализов + `u["state"] = S_LABS`
7. **`cmd_next` (или `handle_nextday`):** проверить `u["day"] == 22` → напоминание о анализах
8. **Команда `/labs`:** зарегистрировать `CommandHandler("labs", cmd_labs)`, реализовать `cmd_labs`
9. **`/skip` НЕ реализуется как команда** — пользователь пишет "пропустить" текстом

### `prompts/4b_labs.txt` (новый файл)

**Входные данные (подставляются в промпт):**
- `{name}` — имя пользователя
- `{diagnosis}` — свободный текст диагноза
- `{labs_raw}` — сырой текст анализов
- `{duration}` — длительность симптомов

**Используемая модель:** `REASON_M` (arcee-ai/trinity-large-preview:free), fallback: `TXT_FB`

**Ожидаемый формат ответа:** plain text (не JSON), прямой ответ пользователю

**Задача промпта:**
- Интерпретировать каждый показатель относительно нормы
- Выделить что критично именно для кожного заболевания
- Скорректировать рекомендации (добавки, питание)
- Тон: понятный, без жаргона, с дисклеймером "не замена врачу"
- Если показатель вне нормы — объяснить что это значит для кожи

## Примеры сообщений бота

### Рекомендация сдать анализы (после фотоанализа)
```
🔬 По твоему диагнозу рекомендую сдать:

📋 Базовые (важны для всех):
• ОАК (общий анализ крови)
• СОЭ и CRP — маркеры воспаления
• Витамин D
• Цинк
• IgE общий
• Гистамин
• Копрограмма
• Анализ на паразитов (лучше 3 раза)
• Дисбактериоз
• Ревматоидный фактор (RF)

⚠️ Дополнительно для акне:
• ДГЭАС, тестостерон, ЛГ/ФСГ (гормоны)
• Глюкоза, инсулин

Уже есть готовые анализы? Пришли фото бланка или напиши значения
(пример: D=18, цинк=7.2, IgE=145)

Нет анализов — напиши "пропустить"
```

### Напоминание в 4-ю неделю (первый раз)
```
🔬 Неделя 4 — время анализов!

Ты прошёл 3 недели программы. Теперь важно сдать анализы,
чтобы оценить прогресс и скорректировать план.

Вот список для твоего диагноза: [список]

Пришли результаты — интерпретирую и обновлю рекомендации.
Или напиши "пропустить" чтобы продолжить.
```

## Ограничения и дисклеймеры

- Рекомендации по анализам носят информационный характер, не заменяют консультацию врача
- OCR фото бланка может ошибаться — пользователь может скорректировать текстом
- При диагнозе melanoma — явный призыв обратиться к дерматологу немедленно
- `/skip` НЕ реализуется как Telegram-команда во избежание коллизий; используется текстовый ввод
